## 第 4 步：设定时（每 5 分钟）—— SQL Editor

回 **SQL Editor**，跑下面这段（一次性）。它用 pg_cron + pg_net 每 5 分钟调一次上面的函数：

```sql
-- 开扩展（第一次要开，已开会跳过）
create extension if not exists pg_cron;
create extension if not exists pg_net;

-- 每 5 分钟触发 remind-cron
select cron.schedule(
  'rsip-remind-cron',
  '*/5 * * * *',
  $$
  select net.http_post(
    url := 'https://mlthrrozzqqumhpqomgn.supabase.co/functions/v1/remind-cron',
    headers := jsonb_build_object(
      'Content-Type','application/json',
      'Authorization','Bearer ' || current_setting('app.service_key', true)
    )
  );
  $$
);
```

> ⚠️ 上面 `current_setting('app.service_key')` 需要你先把 service_role key 存成数据库设置，或直接把 Bearer 换成明文 service_role key。**更简单的做法**见下方「第 4 步·简版」。

### 第 4 步·简版（推荐，省事）

如果懒得配 `app.service_key`，直接把 service_role key 写进 SQL（它只存在你自己项目的 cron 定义里，不外泄）：

1. Project Settings → API → 复制 **service_role** key（`eyJ...` 一长串，**保密**）。
2. SQL Editor 跑：

```sql
create extension if not exists pg_cron;
create extension if not exists pg_net;

select cron.schedule(
  'rsip-remind-cron',
  '*/5 * * * *',
  $$
  select net.http_post(
    url := 'https://mlthrrozzqqumhpqomgn.supabase.co/functions/v1/remind-cron',

      'Authorization','Bearer 这里粘贴你的_service_role_key'
    )
  );
  $$

```

改了主意想停：`select cron.unschedule('rsip-remind-cron');`
看有没有在跑：`select * from cron.job;`

## 第 5 步：前端订阅一次（在手机上）

1. 手机把 App「添加到主屏幕」，从主屏幕图标打开。
2. 进去开任意一条提醒 / 或点铃铛开每日汇总 → 系统弹授权 → 允许。
   （这一步会把这台设备的推送凭据写进 `push_subscriptions` 表。）
3. 回控制台 Table Editor 看 `push_subscriptions` 有没有多出一行 —— 有就成了。

之后设个 1～2 分钟后的提醒时间，把 App 划掉（后台），到点应该收到推送。

## 排查

- **没收到推送**：先看 `push_subscriptions` 有没有你这台设备的行；再看 Edge Function 日志（函数详情页 Logs）有没有报错；确认 secret 三个都填了、`remind-cron` 手动 invoke 返回 `ok`。
- **收到重复**：正常情况下同一提醒一天只推一次（`last_sent` 去重）。
- **时间不对**：前端订阅时会把你的时区写进 `sync_state.data.notify.tz`，服务端按它算。换时区后重新打开一次 App 即可刷新。
- **iOS 限制**：必须 iOS 16.4+、必须从主屏幕图标（standalone）打开，Safari 标签页收不到——这是苹果的硬限制。
